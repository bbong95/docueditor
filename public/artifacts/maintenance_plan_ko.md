# 🔧 하자보수 계획서 (Maintenance Plan)
**문서 번호:** MAINT-v49.0 | **작성일:** 2026-02-14 | **버전:** 1.0

## 1. 개요 (Overview)
본 계획서는 **DocuEditor** 시스템의 안정적인 운영을 위해, 인도 후 발생할 수 있는 결함(Bug)의 수정 및 기능 개선 프로세스를 정의합니다.

## 2. 유지보수 대상 (Scope)
*   **대상 시스템**: DocuEditor Web Application (v49.0 기준)
*   **유지보수 기간**: 검수 완료일로부터 12개월 (무상)
*   **지원 범위**:
    *   소프트웨어 버그 및 오류 수정
    *   브라우저 버전 업데이트에 따른 호환성 패치
    *   기존 기능의 성능 최적화 (OCR 속도 등)

## 3. 유지보수 절차 (Process)

### 3.1 이슈 접수 (Issue Reporting)
1.  사용자 또는 운영자는 GitHub Issues 또는 이메일을 통해 이슈를 등록합니다.
2.  필수 기재 사항: 재현 경로, 스크린샷, 사용 브라우저 버전, 에러 로그.

### 3.2 분류 및 우선순위 선정 (Triage)
| 등급 | 정의 | 처리 목표 (SLA) |
|:---:|:---|:---|
| **Critical** | 서비스 사용 불가 (무한 로딩, 크래시) | 4시간 이내 착수, 24시간 내 해결 |
| **Major** | 핵심 기능(OCR, PPTX 저장) 오류 | 24시간 이내 착수, 3일 내 해결 |
| **Minor** | UI 깨짐, 오탈자 등 사용상 지장 없음 | 주간 정기 배포 시 수정 |

### 3.3 수정 및 배포 (Fix & Deploy)
1.  개발 서버(Local)에서 이슈 재현 및 원인 분석.
2.  수정 코드 작성 및 Unit Test 수행 (`npm test`).
3.  Staging 환경 배포 및 QA 검증.
4.  Production 환경 배포 (`npm run build` -> Deploy).

## 4. 정기 점검 계획 (Regular Checkup)
*   **월간 점검**: 종속성 라이브러리(Dependencies) 보안 취약점 점검 (`npm audit`).
*   **분기 점검**: Tesseract Core 및 AI 모델 업데이트 검토.
