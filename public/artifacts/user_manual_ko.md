# 📘 DocuEditor v2.0 공식 사용자 가이드 (Official User Guide)

> **CodeWiki Level Documentation**
> 본 문서는 **DocuEditor v2.0**의 모든 기능과 워크플로우를 포괄하는 엔터프라이즈급 사용자 매뉴얼입니다. "분산형 에이전트 시스템"이 적용된 최신 인터페이스를 기준으로 작성되었습니다.

---

## 1. 인터페이스 개요 (Interface Overview)

DocuEditor는 사용자의 작업 흐름을 최적화하기 위해 **"시네마 스테이지(Cinema Stage)"** 레이아웃을 채택했습니다. 중앙의 콘텐츠 영역을 중심으로 4명의 전문 AI 에이전트가 상단 헤더에 상주하며 사용자를 지원합니다.

### 1.1 메인 스튜디오 화면
![DocuEditor Studio Interface](./main_interface_retry_1771069052188.png)

| 영역 | 명칭 | 기능 설명 |
| :--- | :--- | :--- |
| **A** | **Agent Squad (상단)** | 작업을 감독하고 실행하는 4명의 AI 에이전트 (Architect, Designer, Developer, Analyst). |
| **B** | **Nav Bar (좌측)** | 페이지 탐색 및 히스토리(실행 취소/복구) 패널 접근. |
| **C** | **Stage (중앙)** | PDF/이미지 편집 영역. 드래그 앤 드롭으로 파일을 업로드합니다. |
| **D** | **Properties (우측)** | 선택된 텍스트 박스의 폰트, 크기, 색상 등을 미세 조정하는 속성 패널. |

## 2. 핵심 워크플로우 (Core Workflow)

### 2.1 문서 업로드 (Upload)
1. **파일 준비**: PDF 또는 이미지(PNG/JPG) 파일을 준비합니다.
2. **드래그 앤 드롭**: 중앙 스테이지 영역에 파일을 끌어다 놓습니다.
3. **에이전트 반응**: 
   - **[Developer]** (오렌지색 에이전트)가 즉시 점프하며 파일을 변환하고 텍스트를 감지(OCR)합니다.
   - 대용량 PDF의 경우 페이지별로 분할되어 좌측 패널에 썸네일로 표시됩니다.

### 2.2 텍스트 편집 (Edit)
1. **박스 선택**: 파란색으로 표시된 텍스트 박스를 클릭합니다.
2. **에이전트 반응**:
   - **[Designer]** (핑크색 에이전트)가 회전하며 편집 모드 진입을 알립니다.
3. **스타일 수정**:
   - 우측 패널에서 폰트(Paperlogy 등), 크기, 정렬을 수정합니다.
   - *팁*: 시스템은 원본 텍스트의 스타일을 95% 이상 자동으로 매칭합니다.

### 2.3 내보내기 및 검수 (Export & QA)
1. **다운로드**: 상단 헤더의 **[PDF 다운로드]** 버튼을 클릭합니다.
2. **에이전트 반응**:
   - **[Architect]** (파란색)와 **[Analyst]** (보라색)가 동시에 점프하며 최종 산출물을 생성하고 검수합니다.
3. **결과 리포트**:
   - 작업이 완료되면 **"품질 검사 리포트(Quality Report)"** 창이 자동으로 팝업됩니다.
   - 정확도, 수정 건수, 보안 점검 결과를 확인하고 파일을 저장하세요.

## 3. 에이전트 스쿼드 상세 가이드 (Meet Squad)

각 에이전트는 고유한 역할(Role)과 전문성(Skill)을 가지고 있습니다.

| 아이콘 | 이름 | 역할 | 전문 분야 | 활성 트리거 |
| :---: | :--- | :--- | :--- | :--- |
| 🧑‍💻 | **Architect** | 리더 (PM) | 프로젝트 총괄, 계획 수립 | **상시 활성 (Always)** |
| 👷 | **Developer** | 엔지니어 | 파일 입출력, OCR 연동 | **업로드 시** |

| 🕵️ | **Analyst** | 검수관 (QA) | 품질 보증, 리포트 작성 | **내보내기 시** |

## 4. 문제 해결 (Troubleshooting)

### Q. 텍스트가 인식되지 않아요.
**A.** 이미지 해상도가 너무 낮을 수 있습니다. 확대해서 캡처하거나 원본 PDF를 사용하세요. Developer 에이전트가 최선을 다해 다시 시도할 것입니다.

### Q. 한글 폰트가 깨져 보입니다.
**A.** 브라우저가 웹폰트(Paperlogy)를 로딩 중일 수 있습니다. 3~5초 후 다시 확인해 주세요.

### Q. 에이전트가 움직이지 않아요.
**A.** 에이전트는 사용자가 **'액션'**을 취할 때만 움직입니다. 아무것도 하지 않을 때는 대기 모드(Idle)로 돌아갑니다.
