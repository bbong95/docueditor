# 📱 화면 설계서 (DocuEditor v2.0)

> [!NOTE]
> 본 문서는 "분산형 에이전트" 레이아웃과 "Analyst 품질 리포트" 대시보드를 중심으로 한 DocuEditor 플랫폼의 고해상도 UI/UX 설계 명세서입니다.

## 1. 전체 레이아웃 (The Stage)
콘텐츠가 주인공이 되고, 에이전트들이 조연으로서 이를 지원하는 **"시네마 스테이지"** 레이아웃을 채택합니다.

### 1.1 헤더 영역 (Agent Loft)
- **높이**: `68px`
- **배경**: 글래스모피즘 (`bg-white/80`, `backdrop-blur-md`)
- **로고**: 좌측 정렬, `240px` 너비 확보.
- **에이전트 스쿼드**:
  - **위치**: 상단 절대 위치 `top: 4px`.
  - **배치**: 로고 우측 `500px` 지점부터 좌측 정렬 클러스터링.
  - **멤버 구성**:
    1. **[Architect]** (`left: 500px`): 리더. 항상 활성 상태 (`jump-spin`).
    2. **[Designer]** (`left: 600px`): 아티스트. `isDesigning` 상태일 때 활성.
    3. **[Developer]** (`left: 700px`): 빌더. `isUploading` 상태일 때 활성.
    4. **[Analyst]** (`left: 800px`): 검사관. `isExporting` 상태일 때 활성.
- **액션 영역**: 우측 정렬. "업로드", "이미지 저장", "PDF 다운로드" 버튼 배치.

### 1.2 메인 작업 공간 (The Canvas)
- **패딩**: `40px` (넉넉한 여백 제공).
- **배경**: 따뜻한 종이 질감 (`#F9F9F7` / `hsla(var(--color-bg-base), 0.5)`).
- **그림자**: 부드럽고 확산된 그림자 (`0 20px 40px -12px rgba(0,0,0,0.1)`).
- **페이지 레이아웃**: 다중 페이지 문서를 위한 수직 컬럼 흐름.

## 2. Analyst 품질 리포트 (The Dashboard)
특정 트리거 발생 시 프로젝트 통계와 품질 지표를 보여주는 모달 대시보드입니다.

### 2.1 트리거 메커니즘 (Trigger)
- **자동**: "PDF 다운로드" 완료 직후 (Analyst가 점프를 멈출 때).
- **수동**: 헤더의 **[Analyst]** 에이전트 클릭 시.

### 2.2 UI 구조 (Modal)
- **규격**: `800px x 600px` (고정 중앙 정렬).
- **헤더**:
  - **제목**: "품질 검사 리포트 (Quality Inspection Report)"
  - **검사관**: Analyst 에이전트 아바타 + "QA 인증됨" 배지.
- **본문 (그리드 레이아웃)**:
  - **블록 A (요약)**: "성공! 12페이지 처리 완료."
  - **블록 B (변경 내역)**:
    - 감지된 텍스트 박스: **42개**
    - 사용자 편집: **15회**
    - 스타일 일치율: **98%**
  - **블록 C (무결성 체크)**:
    - [x] 폰트 호환성 (Paperlogy)
    - [x] 레이아웃 무결성
    - [x] 이미지 해상도 (High-Res)
- **푸터**:
  - **주요 액션**: "인증된 PDF 다운로드" (대형 파란색 버튼).
  - **보조 액션**: "상세 로그 보기".

## 3. 마이크로 인터랙션 (Micro-Interactions)
- **에이전트 호버**: `1.1배` 확대 및 바운스 효과.
- **텍스트 박스 포커스**:
  - 테두리: `2px solid Primary`.
  - 배경: 틴트(Tint) 적용.
  - 선택 시 "Pop" 애니메이션.
- **토스트 알림**:
  - 우측 하단 스택(Stacking).
  - Lucide 아이콘을 활용한 미니멀 디자인.
