# 시스템 요구사항 분석 (Analysis)

본 문서는 Google AI Studio와 유사한 기능을 갖춘 AI 기반 애플리케이션 개발 환경의 요구사항을 정의합니다.

## 1. 개요
사용자가 자연어로 앱의 기능을 설명하면, AI가 실시간으로 코드를 생성하고 이를 즉시 미리보기(Preview)할 수 있는 통합 환경을 구축합니다.

## 2. 핵심 기능 요구사항
- **AI 어시스턴트**: 자연어 명령을 코드로 변환 (Gemini 모델 활용).
- **실시간 프리미엄 에디터**: Monaco Editor 기반의 코드 편집 환경.
- **라이브 프리뷰**: 생성된 코드를 즉시 렌더링하여 확인.
- **멀티 티어 OCR**: Tesseract.js 및 멀티 티어 Gemini Fallback을 통한 텍스트 추출.
- **3D 인터랙티브 워크스페이스**: 기존 `AgentOffice3D` 모듈을 재사용하여 몰입감 있는 오피스 환경 제공.
- **산출물 대시보드**: 개발 과정에서 발생하는 각종 문서와 지표를 시각화.

## 5. 주요 기술적 성과 (Phase 2)
### 5.1 정밀 텍스트 제어 엔진 (Precision Text Engine)
- **자유 이동 (Drag & Move)**: 캔버스 줌 배율(`currentScale`)에 독립적인 절대 좌표 보정 알고리즘 구현.
- **다차원 정렬 (Alignment)**: Canvas API의 `textAlign` 및 `textBaseline`을 활용한 9개 구역(Grid) 기반 정렬 안정화.
- **동적 스케일링 (Dynamic Scaling)**: 폰트 크기(`fontSize`)를 확대/축소 배율에 실시간 동기화하여 시각적 일관성 유지.

### 5.2 캔버스 네비게이션 시스템
- **Space-Panning**: 무거운 이미지 환경에서도 부드러운 핸드 툴(Pan) 이동 로직 적용.
- **유연한 상호작용**: '이동-선택-수정'의 상태 전이를 물리적 커서 피드백(grabbing, crosshair)과 결합.

## 6. 기술 부채 및 로드맵
- [ ] **성능 최적화**: 레이어 분리 및 오프스크린 캔버스를 통한 대역폭 절감.
- [ ] **멀티 유저 협업**: 텍스트 수정 내역 실시간 공유 프로토콜 연구.
- [ ] **고도화된 레이어 관리**: 텍스트 박스 간 계층(Z-Index) 조정 UI 추가.

## 3. 기술적 제약 사항 (대 원칙)
- **Zero-Cost**: 모든 인프라 및 모델 연산은 무료 티어 내에서 해결.
- **한글 무결성**: 한글 깨짐 방지를 위해 `Pretendard` 폰트 및 OCR 최적화 적용.
- **CI/CD 구조**: 분석-설계-개발-테스트-QA-배포의 엄격한 파이프라인 준수.

## 4. 사용자 경험 (UX)
- **Rich Aesthetics**: 글래스모피즘, 비비드한 컬러 팔레트, 미세 애니메이션 적용.
- **Excalidraw 구성도**: 모든 시스템 구성도는 Excalidraw 형식을 통해 직관적으로 제공.
